## [0.1.0] — 2026-06-21

Initial release of the standalone HYROX performance skill repository.

### Added

- **27 HYROX coaching skills** under `skills/`, each with `SKILL.md`
  (trigger-precise frontmatter, structured workflow, explicit non-goals,
  default structured output, safety/scope, incomplete-information handling, and a
  Related Skills and Routing section), `examples/` (≥3 worked examples), and
  `evals/` (`cases.json` with ≥10 cases incl. ≥1 adversarial, `rubric.md`,
  `expected-behaviors.md`). 306 eval cases total, 64 adversarial.
- **Pack manifest** (`manifest.json`, id `hyrox-performance`) listing all 27
  skill ids, the routing guide, example workflows, safety behavior, rule-version
  status, and the non-affiliation notice. Neutral display name "Fitness Racing
  Performance Pack" used absent brand authorization.
- **Deterministic math** in `lib/`: `goal-time-model.ts` (split/goal-time model
  for the fixed 8-run/8-station format with compromised-running decay) and
  `split-analysis.ts` (planned-vs-actual comparison + pace-decay). No network, no
  clock, no telemetry. 22 unit tests in `tests/`.
- **16 domain references** under `references/` (coaching principles, race
  structure, running and compromised running, station taxonomy, sled/erg
  strategy, pacing, transitions, simulation design, doubles/relay strategy,
  tapering, athlete intake, safety-and-referral) plus the empty, versioned
  `source-version-registry.json`.
- **Validation, build, and eval tooling** in `scripts/` run via `package.json`:
  `validate` (frontmatter, structure, internal links, trigger-precision lint,
  duplicate detection, source-version checks), `build` (inventory + seed),
  `test`, and `evals` (offline static eval-suite harness with pack-wide
  adversarial-theme coverage).
- **Canonical seed source** `skill-inventory.json` (generated by
  `build-inventory.ts`) and the idempotent **`seed/seed.sql`** (generated by
  `build-seed.ts`) that upserts the pack and skills into the Skill Me catalog.
- **Versioned rules framework**: rules-dependent skills require a season/event,
  use only matching registry versions, warn on staleness, and never scrape or
  embed proprietary HYROX content.
- **Trademark / non-affiliation policy**: `DISCLAIMER.md`,
  `assets/brand/brand-authorization.json` (authorized: false), and validation
  that refuses to label the pack official/verified without complete
  authorization metadata.
- Project docs (`README.md`, `CONTRIBUTING.md`, `SECURITY.md`, `PACK.md`, and
  the `docs/` set: architecture, installation, skill-authoring-standards,
  evaluation-methodology, safety-and-scope, branding-and-trademarks,
  updating-rules-and-standards), MIT `LICENSE`.

### Schema dependency

- The `seed/seed.sql` columns were confirmed against the live skillshelf Supabase
  `skills`/`packs` schema on 2026-06-21. The upsert conflict target is the unique
  `slug` (the `id` PK is a server-generated uuid); `category` maps to the
  `personal` enum bucket with the HYROX domain category in `subcategory`.
  Re-confirm before targeting any other project.
