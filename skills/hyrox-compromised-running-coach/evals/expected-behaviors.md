# Expected Behaviors — Compromised Running Coach

## When this skill should trigger
- An athlete or coach asks to fix the **run immediately after a station**: legs
  dead off the sleds, the slow first 100–200 m out of the roxzone, collapsed
  cadence, panicked breathing, posture, or pace control in the early metres of a
  compromised run — and supplies (or can supply) which stations fade their run
  worst and any recent compromised-run experience.
- "I fall apart on the run after the sleds," "how do I run off heavy legs," "my
  turnover dies leaving the station," "train running tired between stations."

## When it should NOT trigger (route instead)
- "Build my aerobic base / threshold / race-pace / a running block" →
  `hyrox-running-development-plan` (this skill trains the transition; that one
  builds the engine). **Resolve this overlap explicitly every time.**
- "What pace on each of the 8 runs on race day / roxzone targets" →
  `hyrox-race-pacing-planner`.
- "Build me a full race simulation" → `hyrox-simulation-builder` (sims stay sparse).
- "Drill out a sled fault / build station capacity" →
  `hyrox-technique-and-fault-analyzer` / `hyrox-station-progression-builder` /
  `hyrox-sled-performance-specialist`.
- "Rank my limiters" → `hyrox-athlete-assessment`.
- Broad/multi-part request → `hyrox-pack-router`.

## Correct handoffs
- Always coach the first-200 m sequence: pace control → cadence → breathing →
  posture, then station-to-run bricks biased to sled-to-run and lunge-to-run.
- When the athlete hasn't said which stations fade worst, ask, and default to the
  costliest transitions with an explicit note that priority shifts on confirmation.
- Reserve full simulations as sparse benchmarks routed to the simulation builder.

## Failure modes to avoid
- **Sim overload.** Never prescribe a full 8-station simulation every session; it
  crowds out the easy volume and quality running that build the engine.
- **Wrong model.** Never treat the fade as a pure fitness problem solved by more
  mileage; never equate gym sled load with race resistance; never "CrossFit with
  running."
- **Chasing fresh pace.** Never coach hitting fresh pace in the first 100–200 m off
  heavy legs; pace control comes first, cadence before stride length.
- **No progression criteria.** Brick count and run-segment size advance on a
  criterion (target compromised pace held, cadence recovered in the planned
  distance), not the calendar.
- **False precision.** Never invent compromised paces for an athlete with no data;
  anchor with a baseline brick.
- **Outcome guarantees.** Never promise the fade disappears; some positive split is
  expected even when trained (even effort, not even pace).
- **Overriding medicine.** Never engineer sled/lunge bricks around a clinician's
  loading restriction; refer out on sharp/worsening pain or rhabdo red flags after
  heavy eccentric work.
- **Scope creep.** Never build the aerobic block, the pacing plan, or a full
  simulation here.
- **False endorsement.** Never present the work as official or brand-approved.
