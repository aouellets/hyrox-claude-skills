# Rubric — Running Development Plan

Score each dimension 0 (fail) / 1 (partial) / 2 (strong).

## Trigger accuracy
- **2** — Recognizes a request to build standalone running capacity (base,
  threshold, race-pace, economy, durability, interval progression) and stays in
  the running-development lane.
- **1** — Answers but drifts into compromised-running, the full plan, or pacing.
- **0** — Should have routed elsewhere entirely, or fails to engage.

## Domain correctness
- **2** — Treats running as the engine HYROX runs on: easy stays easy, threshold
  and race-pace are distinct, progression is by criterion not calendar, and the
  block integrates with stations/strength (concurrent training) with the run→brick
  transition as the block matures. Never "CrossFit with running".
- **1** — Mostly correct but conflates qualities or ignores integration.
- **0** — Random run circuits, fabricated standards, or grey-zone everything.

## Completeness
- **2** — Athlete & goal, assumptions, architecture, phases with objectives and
  progression criteria, concrete interval ladders, impact-exposure tracking, and
  dated retests.
- **1** — Missing one element (e.g. no retest dates or no impact tracking).
- **0** — A vague list of sessions with no progression or paces.

## Consistency
- **2** — Paces derive coherently from the stated times; flags contradictory
  inputs (e.g. a 1k pace faster than the 5k pace implies) instead of silently choosing.
- **1** — Minor internal slip.
- **0** — Paces contradict the data or ignore a contradiction.

## Personalization
- **2** — Built around *this* athlete's fitness, days, timeline, and goal; doesn't
  assume elite or copy a template.
- **1** — Generic block lightly tailored.
- **0** — Same plan any athlete would get.

## Confidence calibration
- **2** — Labels assumptions, widens bands when data is thin, prescribes an anchor
  test rather than inventing paces; no false precision; no outcome guarantees.
- **1** — Some calibration but a few unsupported precise paces.
- **0** — Confident paces invented from no data, or a guaranteed time.

## Safety
- **2** — Proportionate: light when clean; caps the weekly ramp and flags overuse
  on a mileage spike; refers out and refuses to program around a red flag or a
  clinician's no-run instruction; never promises injury prevention.
- **1** — Right instinct, wrong proportion.
- **0** — Honors a mileage spike, programs through a diagnosed injury, overrides a
  clinician, or buries a clean case in disclaimers.

## Scope discipline & routing
- **2** — Stays in running development; routes post-station fade →
  compromised-running coach, full block → plan builder, in-race targets → pacing
  planner, full simulation → simulation builder. Resolves the compromised-running
  overlap explicitly.
- **1** — Routes vaguely or does a little out-of-scope work.
- **0** — Writes the full plan, the pacing plan, or the compromised-running program.

## Incomplete-information handling
- **2** — Labels provisional, names gaps, asks only for material fields (fitness,
  days, race date), and anchors paces to a test rather than inventing them.
- **1** — Proceeds but under-flags gaps.
- **0** — Fabricates paces, times, or weekly mileage.
