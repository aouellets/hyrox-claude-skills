# Rubric — Wall Ball Completion Planner

Score each dimension 0 (fails), 1 (partial), 2 (meets the bar).

## Fatigue realism (the defining dimension)
- **2** — Plans explicitly for **cooked legs at the end of the race**, sizes the
  working set to *fatigued* capacity (not fresh), builds a **broken-set plan with
  planned and backup breaks**, and **refuses an unbroken-fresh-only set** even on
  request — explaining that planned breaks beat a stall.
- **1** — Mentions fatigue but still anchors the plan on the fresh number or omits
  backup breaks.
- **0** — Prescribes an unbroken set built on fresh capacity; ignores end-of-race legs.

## Rep-count discipline
- **2** — **Requires the season/division rep count or uses a clearly-labeled
  assumption**, never asserts it from memory, routes to
  `hyrox-division-and-rules-advisor`, and gives the set structure as proportions/method
  so it holds for any count.
- **1** — Notes the count varies but still leans on a remembered number.
- **0** — States a specific season/division rep count as fact from memory.

## Domain correctness
- **2** — Squat/throw efficiency (depth + target standard, catch-into-squat rhythm,
  leg-driven throw), planned descending sets with breaks, breathing tied to the rep
  cycle, no-rep mitigation (depth over speed late), and **pre-fatigued** practice.
- **1** — Mostly correct, misses an element (e.g. no-rep mitigation or pre-fatigued
  practice).
- **0** — Generic "do wall balls" advice; trains only fresh; ignores no-reps.

## Completeness
- **2** — Framing+count, fatigued capacity estimate, efficiency, broken-set plan with
  backup breaks, breathing, no-rep mitigation, pre-fatigued practice + criterion +
  retest, integration.
- **1** — Missing a section or the progression criterion/retest.
- **0** — Loose advice, no structure.

## Personalization
- **2** — Keyed to the athlete's count, fresh/fatigued numbers, degradation pattern,
  and timeline.
- **1** — Plausible but generic.
- **0** — Ignores the inputs.

## Incomplete-information handling
- **2** — Provisional; **rep count missing → labeled assumption + rules-advisor
  pointer + proportional plan**; **capacity missing → pre-fatigued baseline test**;
  flags that a fresh-only number is not the race set; never invents.
- **1** — Provisional but fabricates a count or a fatigued number.
- **0** — Asserts a count or invents a working set as fact.

## Consistency
- **2** — Surfaces contradictions (unbroken demand vs a low fresh max) and resolves to
  a realistic broken plan.
- **0** — Silently programs the contradiction.

## Safety
- **2** — Proportionate scope note; knee/shoulder red flags (giving-way, sharp or
  worsening pain) override the request and route to a professional; no
  prevention/outcome promise.
- **1** — Mentions safety but plans around the symptom.
- **0** — Plans around a red flag or promises an outcome/completion time.

## Scope discipline / routing
- **2** — Routes squat strength → progression builder, rep count → rules advisor,
  whole-race pacing → pacing planner, vague "what's wrong" → fault analyzer, with clear
  handoffs.
- **0** — Absorbs another skill's job or asserts a rules-owned count.
