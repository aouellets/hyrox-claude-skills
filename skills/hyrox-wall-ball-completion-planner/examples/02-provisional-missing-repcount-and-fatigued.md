# Example 2 — Incomplete data: rep count unknown, only a fresh number

**User:**
> How should I break up the wall balls? I can do about 30 in a row when I'm fresh.

**Response:**

This is a **provisional** wall ball plan. I can give you the method and the breathing
now, but two gaps change the actual sets — and I won't fake either one.

## What I'm missing (and why it matters)
1. **Your season/division rep count.** The total changes by season and division, and
   **I won't state it from memory.** Get the authoritative number from
   `hyrox-division-and-rules-advisor`. I'll build the plan as **proportions** so it
   drops onto any count.
2. **A *fatigued* capacity.** Your "30 fresh" is **not** your race number. Wall balls
   are the last station on cooked legs — the race set is meaningfully smaller. Without
   a tired test I'd be guessing the size of the cliff.

## Capacity read (labeled estimate)
From 30 fresh, a realistic **fatigued first set is ~10–12** — plan under the cliff, not
at it. This is an estimate until you do a pre-fatigued test.

## Squat & throw efficiency (actionable now)
- Hit **depth and target every rep** — a no-rep is a redo on the worst legs of the day.
- **Catch into the squat**, don't catch-pause-squat. Drive with the legs; don't
  arm-muscle when the legs fade.

## Broken-set method (proportional, fill the count later)
Plan **descending sets with planned breaks**, e.g. start ~12, then settle to sets of
8, then 6–7 as fatigue builds, short 3–5-breath breaks between. **Pre-permit a few
backup breaks** so stopping is the plan, not a failure. Once you give me the rep count
and a fatigued test, I'll write the exact set list (like 12/10/8/8/…).

## Breathing
Exhale on the throw; use each planned break for a few deliberate breaths to reset.

## Baseline test (do this first)
Pre-fatigue yourself (leg work or a hard 400 m) → wall balls to a controlled stop.
Note the number before depth degrades. That's your real planning input.

**Send me the rep count and your fatigued number and I'll turn this into exact sets and
a retest date.** The method above doesn't change.
