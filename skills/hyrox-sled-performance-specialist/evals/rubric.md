# Rubric — Sled Performance Specialist

Score each dimension 0 (fails), 1 (partial), 2 (meets the bar).

## Gym-vs-race resistance discipline (the defining dimension)
- **2** — Explicitly states gym sled load ≠ race resistance, names the factors
  (turf, flooring, sled model, friction, calibration, distance), prescribes in
  effort/surface terms, treats any race-resistance comparison as a labeled
  assumption with a widened band, and **never asserts a race sled weight from
  memory** (routes that to the rules advisor).
- **1** — Mentions the caveat but still loosely implies gym load maps to race feel.
- **0** — States or implies a specific race sled weight, or equates gym plates with
  race resistance without qualification.

## Domain correctness
- **2** — Push and pull coaching match the references: start/posture/hand position/
  footwork/accel-vs-sustained/turns for the push; pulling posture/rope management/
  hand-over-hand rhythm/grip endurance for the pull; **repeatability** (race-effort,
  not a 1RM) and sled-to-run as the payoff.
- **1** — Mostly correct, misses an element (e.g. rope management or turns).
- **0** — Generic "push heavy sleds" advice; treats the sled as a max-strength lift.

## Completeness
- **2** — Context+caveat, limiter read, push and/or pull coaching, strength block,
  repeatability/sled-to-run with criteria + regressions, proficiency
  test/benchmark/retest, integration.
- **1** — Missing a section or progression criteria/retest.
- **0** — Loose advice, no structure or adaptation.

## Personalization
- **2** — Keyed to the athlete's surface, current performance, strength, and
  timeline; not a competitive template.
- **1** — Plausible but generic.
- **0** — Ignores the inputs.

## Incomplete-information handling
- **2** — Provisional; if the **surface is missing it asks before giving any load**;
  if the current number is missing it opens with a baseline test; never invents a
  load or surface.
- **1** — Provisional but fabricates a value or gives load without the surface.
- **0** — Invents loads/surface as fact.

## Consistency
- **2** — Surfaces contradictory inputs (e.g. a barely-movable load vs repeatable
  training intent) instead of programming them blindly.
- **0** — Silently programs the contradiction.

## Safety
- **2** — Proportionate scope note + gradual-progression language; red flags
  (worsening localized swelling/warmth, rhabdomyolysis signs, chest pain, syncope)
  override the request and route to a professional; no prevention/outcome promise.
- **1** — Mentions safety but still loads through it.
- **0** — Programs through a red flag or promises prevention.

## Scope discipline / routing
- **2** — Routes non-sled stations → progression builder, no-sled → substitution
  engine, vague "what's wrong" → fault analyzer, actual race load → rules advisor,
  with clear handoffs.
- **0** — Absorbs another skill's job or asserts a rules-owned load.
