# Expected Behaviors — Roxzone And Transition Strategist

## When this skill should trigger
- An athlete or coach asks to sharpen the **roxzone / transitions**: station
  approach and exit, route awareness, where to ease and resume pace, equipment
  entry and setup, chalk, planned hydration, walk-vs-run choices, rower strap
  handling, partner handoff cues, or transition target ranges — and supplies (or
  can supply) which transitions leak time and any venue or measured-transition data.
- "My roxzones are slow," "how do I transition faster," "the row straps cost me,"
  "should I walk or run through the roxzone," "fix my route awareness."

## When it should NOT trigger (route instead)
- "What pace each run / station target times / the fade ladder / effort ceilings" →
  `hyrox-race-pacing-planner`. **Resolve the overlap: the pacing planner owns the
  runs/stations and sets transition target ranges in the whole plan; this skill
  drills the transition execution and refines those ranges.**
- "Who does which station reps / the run split in doubles/relay" →
  `hyrox-doubles-strategy-builder` / `hyrox-relay-strategy-builder` (this skill
  covers only handoff communication/mechanics).
- "Handoff legality / division specifics" → `hyrox-division-and-rules-advisor`
  (season-dependent; never from memory).
- "Fix my rowing stroke / 1000m pacing" → `hyrox-erg-strategy-coach`.
- "Run a full timed simulation" → `hyrox-simulation-builder`.
- Broad/multi-part request → `hyrox-pack-router`.

## Correct handoffs
- Always frame the roxzone as **16 measurable, trainable transitions**; give a
  transition-by-transition plan with station-specific walk-vs-run and target ranges
  labeled as assumptions to measure.
- When venue or measured data is missing, give universal principles plus an
  assumption frame and route a timed simulation and a venue walkthrough.
- For doubles/relay, build the handoff **communication and mechanics** and route the
  work allocation and rules out.

## Failure modes to avoid
- **Blanket walk/run rule.** Never give "always run" or "always walk"; make it
  station-specific.
- **Fabricated layout or times.** Never invent a specific venue's roxzone layout or
  a "standard" per-transition second-count; label ranges as assumptions to measure.
- **Scraping.** Never scrape HYROX's website or venue maps; ask for the athlete's
  walkthrough notes or licensed/official briefing.
- **Asserting rules from memory.** Handoff legality is season-dependent; route to
  the rules advisor.
- **Scope creep.** Never build the run/station pacing, allocate partner work, or
  write the rowing stroke plan here.
- **Pushing through pain.** Never load a sharp, worsening grip/forearm or joint pain
  harder; regress and refer out.
- **False endorsement.** Never present the plan as official or HYROX-approved; state
  non-affiliation when pressed.
