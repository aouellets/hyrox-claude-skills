# Security Policy

This repository is **content and tooling only**: Markdown skills, JSON
manifests, deterministic TypeScript math (`lib/`), Node/TS validation and build
scripts (`scripts/`), and a generated SQL seed (`seed/seed.sql`). It ships no
server, no runtime service, and no secrets.

## Data handling and privacy

- The `lib/` math (`goal-time-model.ts`, `split-analysis.ts`) makes **no network
  calls**, reads no clock, and **collects, persists, or transmits no athlete
  data**. Inputs go in, numbers come out.
- The skills are coaching content. When a skill is used inside a model, any
  athlete data the user provides stays within that session; nothing in this repo
  exfiltrates it.
- No analytics or telemetry are embedded anywhere in this repository.

## The SQL seed

`seed/seed.sql` is an idempotent upsert generated by `scripts/build-seed.ts`. It
contains only the public skill content and pack metadata in this repo. Before
applying it:

- Review it. Run it against the intended database only.
- Re-confirm the target schema (see the header in `seed/seed.sql` and
  `docs/architecture.md`); column names were confirmed against one specific
  project and may drift.
- Replace the `OWNER` placeholder in the repository URL before publishing.

## Reporting a vulnerability or content concern

If you find a security issue in the tooling, or a safety/accuracy problem in the
skill content (for example, guidance that could be unsafe, or a missing referral
where a red-flag symptom should be escalated), please open a GitHub issue, or for
sensitive reports use the repository's private security advisory channel if one
is configured. Include steps to reproduce and the affected file path.

We treat **unsafe coaching guidance** (e.g. content that encourages training
through red-flag symptoms, or that promises injury prevention or guaranteed
outcomes) as a defect on par with a code vulnerability. See
`references/safety-and-referral.md`.

## Supported versions

This is a content repository released under SemVer (see `CHANGELOG.md`). Fixes
are applied to the latest release.
